TelaGallery - Практична робота №1
*Проблема*:
Багато користувачів зберігають візуальні ідеї хаотично: у папках, месенджерах або закладках браузера. Це ускладнює пошук, структурування та повторне використання матеріалів. Виникає потреба у простому інструменті для збереження та організації візуального контенту.

*Пітч*:
TelaGallery - це вебплатформа для збереження та організації візуальних ідей у персональні галереї. Продукт допомагає студентам, дизайнерам і творчим користувачам швидко зберігати зображення та впорядковувати їх у тематичні колекції. Користувач отримує зручний простір для натхнення без зайвих дій і складних налаштувань.

**Ціль першої ітерації**
*Створити мінімально життєздатний продукт (MVP), який дозволяє:*
- зареєструватися,
- створити галерею,
- додати зображення,
- побачити його у списку галереї.

**MVP**
 *Головний сценарій:*
- Користувач заходить на платформу,
- створює галерею,
- додає зображення,
- бачить його у своїй колекції.
Без цього сценарію цінність продукту не реалізується.

**Користувацький сценарій (послідовність дій)**
 *Умови початку:*
Користувач відкрив сайт і не має збережених галерей.

 *Кроки:*
- Натискає кнопку "Створити галерею".
- Вводить назву.
- Натискає "Зберегти".
- Натискає "Додати зображення".
- Завантажує файл.
- Після завантаження бачить його у галереї.

 *Очікуваний результат:*
Зображення відображається у створеній галереї без помилок і додаткових дій.

**Карта стейкхолдерів**
  Користувачі - отримують основну цінність (збереження ідей).
  Викладач / замовник - оцінює досягнення цілей і простоту реалізації.
  Розробник - відповідає за реалізацію в межах часу та ресурсів.

 *Очікування:*
- проста взаємодія
- мінімальна кількість кроків
- стабільність базових операцій

 *Обмеження:*
- обмежений час розробки
- мінімальні ресурси
- відсутність складної інфраструктури

**Критерії приймання MVP**
- Головний сценарій виконується не більше ніж за 1-2 хвилини.
- Відсутні критичні помилки під час додавання зображення.
- Нова людина може виконати сценарій без пояснень.
- Проста операція працює стабільно протягом тестового тижня.

**План перевірки гіпотез**
  Гіпотеза: користувачам важливо швидко зберігати візуальні матеріали без складної структури.
 *Спостереження:*
- Чи користувач завершує головний сценарій?
- Чи виникають труднощі при створенні галереї?
- Чи потрібно скорочувати кількість кроків?
  Метод перевірки:
- Спостереження за виконанням сценарію
- Збір короткого відгуку після першого використання

**Основні невизначеності**
- Чи достатньо однієї галереї на початковому етапі?
- Чи потрібна система тегів у першій версії?
- Чи впливає дизайн інтерфейсу на швидкість виконання дії?
  Для зменшення ризиків MVP включає лише базовий функціонал без додаткових налаштувань.
  
**Definition of Success**
  Перша ітерація вважається успішною, якщо:
- Користувач без підказок створює галерею і додає зображення.
- Процес не викликає критичних помилок.
- Час виконання сценарію не перевищує 2 хвилини.
- Отримано щонайменше 2 позитивні відгуки про простоту використання.

**Наступні кроки**
- Додати можливість редагування галерей.
- Додати сортування або категорії.
- Провести повторну перевірку з оновленою версією.
